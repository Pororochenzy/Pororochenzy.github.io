Redis安装

当前redis最新稳定版本是4.0，常用版本3.2版本。

step1:下载

wget http://download.redis.io/releases/redis-3.2.8.tar.gz

step2:解压

tar -zxvf redis-3.2.8.tar.gz

step3:复制，放到usr/local/redis目录下

sudo mv ./redis-3.2.8 /usr/local/redis/

step4:进入redis目录

cd /usr/local/redis

step5:生成

  1. 安装c语言编译器gcc
    sudo apt-get install gcc
  2. 安装编译命令make
    sudo apt-get install make(这一步可能会出问题，根据提示执行命令)
  3. 生成
    sudo make(比较慢)
    我们现在做的是源码安装 ，如果用apt-get用依赖包管理 下载的话 就不能去做一下文件配置，配不了集群
    redis 的gcc 是为了编译我们下载下来的redis源码
    redis源码因为是c写的嘛
    然后可以观察到，/usr/local/redis$ sudo vim Makefile
    它这个文本里面就 写的一些关于ubutu在linux的 一系列安装命令，它帮你报装好在一起，然后你用make 去
    执行

    step6:测试,这段运行时间会比较慢

    sudo make test

tcl8.6.1-src.tar 相当于类库 ，redis要用到的库

解决方法

  1. wget http://downloads.sourceforge.net/tcl/tcl8.6.1-src.tar.gz
  2. sudo tar xzvf tcl8.6.1-src.tar.gz -C /usr/local/
  3. cd /usr/local/tcl8.6.1/unix/
  4. sudo ./configure
  5. sudo make（时间比较长）
  6. sudo make install
    注意：上面是在/usr/local/tcl8.6.1/unix/里进行make install

step7:安装,将redis的命令安装到/usr/local/bin/目录

在/usr/local/redis 执行sudo make install
sudo make install （时间比较长）

step8:安装完成后，我们进入目录/usr/local/bin中查看

  • redis-server redis服务器
  • redis-cli redis命令行客户端
  • redis-benchmark redis性能测试工具
  • redis-check-aof AOF文件修复工具
  • redis-check-rdb RDB文件检索工具

step9:配置文件，移动到/etc/目录下

配置文件目录为/usr/local/redis/redis.conf
sudo cp /usr/local/redis/redis.conf /etc/redis/

##配置

Redis的配置信息在/etc/redis/redis.conf下。

查看

sudo vi /etc/redis/redis.conf

核心配置选项

绑定ip：如果需要远程访问，可将此行注释，或绑定一个真实ip

bind 127.0.0.1

端口，默认为6379

是否以守护进程运⾏

a)如果以守护进程运⾏，则不会在命令⾏阻塞，类似于服务
b)如果以⾮守护进程运⾏，则当前终端被阻塞
c)设置为yes表示守护进程，设置为no表示⾮守护进程
d)推荐设置为yes
daemonize yse 使它变为后台进程，前台进程的话 会卡死在黑窗口，例如启动GoLand
–另外，不小心按了ctrl+Z 怎么办 –>直接按 fg
daemonize yes

数据文件

dbfilename dump.rdb

数据文件存储路径

dir /var/lib/redis

日志文件

logfile /var/log/redis/redis-server.log

数据库，默认有16个

database 16

主从复制，类似于双机备份

slaveof

##服务器端和客户端命令

服务器端的命令为redis-server

可以使用help查看帮助文档

redis-server –help

个人习惯

注意，在安装编译好redis时，系统会默认以服务的形式去启动redis-server, redis服务的启动方式有2种，一种是直接服务的方式管理redis服务
假如redis以服务时的形式启动的话，那么用kill -9 XX不能够杀死的，终止那个进程后，它又会以另一个端口启动

不推荐使用服务的方式管理redis服务

启动 sudo service redis start
停止 sudo service redis stop
重启 sudo service redis restart

重点重点，不用服务方式开启redis，那么如何改用手动的方式去启动呢

客户端

客户端的命令为redis-cli

连接redis

控制台窗口 输入 redis-cli 就能进入redis了
另外补充,如果想之后显示中文字符的话用, redis-cli –raw

运行测试命令


切换数据库
数据库没有名称，默认有16个，通过0-15来标识，连接redis默认选择第一个数据库
select n
